分层管理

从上到下

  • 应用层：负责处理像 http 这样的报文，直接对口应用程序
  • 传输层：负责处理 TCP 这样的可靠连接，通过三报文，端口的方式确保传输的报文完整，对口上层应用层
  • 网络层：也就是 IP 层，通过发送一定大小的数据报来传送传输层给到的数据，不保证可靠，但会尽力发送更多的报文
  • 链路层：负责管理 IP 地址与各种网络实体设备的映射，确保找到正确，最短的传输路径

URI 与 URL

URL

URL是最常见的地址，他叫统一资源定位符，是Web浏览器的通用地址格式

URI

统一资源标识符，常见的有 http，ftp，mailto，telent，file

它有一个统一的格式

  • 协议不用说常见的就那几种
  • 登录信息不常用
  • 服务器地址必须，可以是域名，也可以是IpV4或IpV6地址
  • 端口号不指定就位默认，比如 http 默认就为 80
  • 文件路径不用多说
  • 查询参数很常用，接问号后
  • #之后一般为一片文章的某个部分，可能是目录，a标签的锚
