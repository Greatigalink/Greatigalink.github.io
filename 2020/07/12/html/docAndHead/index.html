文档类型与头部元素

文档类型

用来告诉浏览器应该按照哪一个版本来解析

html4 及以前的版本是基于SGML（标准通用标记语言/Standard Generalized Markup Language），因此必须引用DTD，例如下面

Html4 的 DOCTYPE

而 html5 不在基于上面的标准，无需引用 DTD，所以定义就变得简洁了

Html5 的 DOCTYPE

  • DOCTYPE 比不是一种标签，而是指令，告诉浏览器用什么版本解析 html
  • 大小写不敏感

头部元素

指内嵌在 head 标签中的元素

title

定义网页标题，很常用，浏览器具栏和收藏夹能看到的网页标题

base

通常用来定义基准地址，元素中的部分地址可以加上这个基准地址变成一个完整的地址

link

很重要，定义外链样式，可以是 css 或者 js 文件，在网页加载时并行加载，所以网页显示时比较完整

script 与 style

定义脚本文件和内联样式，它的加载会在网页加载完后才开始

meta

另一个特别重要的元素，定义规范了网页的许多东西，提供了元信息

content

定义与 name，http-equiv 有关的信息

name 属性

可以定义一些关键词，利于搜索引擎搜索

  • 常用 author，keywords，description

http-equiv

向浏览器发送一些头部信息，指定某些规范或者适配什么，具体的还要根据内容来确定其作用

服务器必须要发送一个：content-type:text/html，告诉浏览器接受一个 html文档
