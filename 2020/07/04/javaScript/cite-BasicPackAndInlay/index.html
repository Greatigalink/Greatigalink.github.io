基本包装类型与内置对象
2020-07-04|JavaScript

基本包装类型

基本包装类型主要针对 String, Boolean, Number，他们虽然是基本类型值，但是也有创建他们的实例方式，

  • 一般情况下，采用字面量的方式定义时，他们就是基本类型值
  • 可以采用 new Object() 或者对应的构造方法创建，此时就是对象

我们知道它们拥有各自的一些方法，但是基本类型值本身只是一个值并没有方法，所以在调用基本类型值的方法时，后台会创建各自的实例并传入他们的值，在调用实例的方法之后又消除实例，回到原本的基本类型值。

看例子

  • 对于另外的 Number 与 Boolean 也是一样，使用字面量直接定义的数据在调用三种类型特有方法时都会进行类似上面的步骤

  • 所以看似这三种类型经过了实例创建这一步，但依然不能像操作对象那样给它添加新属性

  • 当然也可以使用 new Object() 的方式创建它，但是不推荐这样，因为容易把基本类型值与引用类型值搞混了

另外一点，虽然转型函数与构造函数很像，但经过他们处理后的值一个是基本类型，一个是引用类型

Boolean

  • 可以使用它的构造函数来创建实例
  • Boolean 重写了valueOf方法，返回基本类型值 true 和 false
  • 重写了 toString() 方法，返回 true 和 false 的字符串

不建议使用Boolean对象，在布尔运算中，对象始终会被转为 true ，如果用构造函数创建了数值为 false的实例，那么比较结果一直都会不在预期内

Number

定义

  • 使用构造函数来定义
  • 它的实例重写了 toSting() 与 toLocalString() ，返回熟知的字符串表示
  • 重写了 vlueOf() ，返回基本类型值的表示

toString() 可以将数值转变为字符创，并且接受一个参数来指定转换为何种进制

方法

toFixed()

指定数值的表示为字符串后的小数数位个数，默认无小数位，看例子

toExponential()

指定将数值表示为指数表示法的字符串，接受一个参数表示小数位的个数

toPrecision()

推荐使用该方法，他会判断一个数值用上述两种方法的哪一种来表示比较合适，接受一个参数表示要显示的位数不包括指数部分

String

  • 拥有 length 属性，长度包含所有的空格

字符方法

本身没有字符类型

charAt()

接受一个参数表示字符的位置，返回字符

charCodeAt()

接受一个参数表示字符的位置，返回字符的编码值

str[i]

可以直接使用 str[0] 的形式来获取字符

操作方法

concat() 和 + 操作符

这两种方法都可以拼接字符串，空格也算，concat 可以接受多个参数，同时拼接多个字符串，但往往使用 + 多一点

ES6引入了占位符概念，很方便就能在字串中添加变量值 ${expression}，前提字符串得用反撇号包起来

slice()

  • 参数1表示起始位置，参数2表示结束位置加1
  • 遇到负数时，先将负数加上字符串总长，按得到的值进行操作

substring()

  • 参数1表示起始位置，参数2表示结束位置加1
  • 遇到负数时，把所有负数按0进行操作
  • 此方法有个特点，若是起始位置与结束位置颠倒，值也是一样的，因为这个方法总是从小的那一个参数开始，大的结束（仅适用于此方法）

    就比如 s.substring(2,5) 与 s.substring(5,2) 结果一样

substr()

  • 参数1表示起始位置，参数2表示需要剪切的字符串长度
  • 遇到负数时，把第一个负参数转换为其值加上字串总长，将第二个负参数转换为0

位置方法

indexOf() 和 lastIndexOf()

与数组那里的差不多，一个从前往后找，一个从后往前找，接受两个参数，参数1表示所找字符，参数2表示从哪开始找；返回字符串位置，没找到返回-1，若是查找一个字符串，则返回该串首字符位置

trim()

可以将字符串前面和后面的空格删除，但是字符串里的空格不会

  • IE 9+，Firefox 3.5+，Safari 5+，Opera 10.5+，Chrome 都支持此函数
  • 另外 Firefox 3.5+ Safari 5+ Chrome8+ 还支持 trimLeft() 与 trimRight()，分别删除前面和后面的空格

大小写转换

由于特别地区转换规则不一致，所以分为通用方法和针对地区的方法

  • 通用方法

toLowerCase() 转小写 和 toUpperCase() 转大写

  • 针对地区方法

toLocaleLowerCase() 转小写 和 toLocaleUpperCase() 转大写

建议使用针对地区的方法

模式匹配

match()

接受一个参数，要么是正则表达式，要么是一个 RegExp 对象，返回一个数组包括匹配到的字符串

search()

查找指定字符串，返回找到的第一个字符串首字符索引，从头开始往后找

replace()

指定要匹配的字串或者正则表达式，并将其替换为指定字符串，分别为参数1和参数2

这里还有一个字符序列方法可以灵活替换指定文本

  • $n 使用找到的第n个字符串, 0 ~ 9
  • $nn 使用找到的第nn个字符串, 00 ~ 99

replace的第二个参数还可以是一个函数，此函数接受3个参数，分别为匹配到的字符串；匹配的字串在整个字串的位置；原始字符串

split()

指定一个分隔符分割字符串，并将每一项组为一个数组，参数1为分隔符也可以为正则表达式，参数2为数组大小

目前 IE8 及其之前版本会忽略捕获组，而 Firefox3.6 以前的版本会在未找到匹配项时，在结果中包含空字符串

localeCompare()

用来比较字符串大小

  • 在字母表中，比较字符串在被比较字串之前返回1，等于返回0，后面返回-1
  • 另外因为该方法的实现不一致，导致返回数值不一定为 1，0，-1，所以建议采用 if(x>0)，if(x<0)，else 的形式

fromCharCode()

它是 charCodeAt() 的逆向方法，给定字符编码返回字符，它是属于 String 的构造函数

内置对象

Global

UrI 的解码与编码

url中可能包含一些非法字符，所以需要通过编码过滤无效的字符，从而让浏览器能接受

  • encodeURI()

该方法不对原本属于url的特殊字符进行编码，比如冒号、正斜杠、问号、井字号

  • encodeURIComponent

会对所有非标准字符编码

  • decodeURI

针对encodeURI()编码的 uri 进行解码，对 encodeURIComponent() 使用可能部分字符无法解码

  • decodeURIComponent()

可以解码任何特殊字符的编码

eval() 方法

接受一个参数，参数中为 JS 代码，解析器解析到 eval() 时会将它当成正常代码，并且它与当前执行环境拥有相同的作用域链

  • 外部定义的变量或者函数，可以在 eval() 里面正常执行
  • 反过来， eval() 里面定义的变量或函数，调用时必须在 eval() 语句后执行，因为只有当解析到 eval() 时，里面的变量和函数才能被创建

不要为任何用户提供 eval() 的方法来输入保存数据，这回导致脚本注入的攻击，比如你给用户提供了 eval() 输入评论的方式，当另一个用户打开评论时，评论里面含有的代码就会被执行，盗窃用户的本地cookie信息，从而盗窃账号

Global对象属性

它具有几乎所有的引用类型构造函数，基本包装类型构造函数，以及特殊值 undefined，NaN，Infinity，以及各种提示错误的的构造函数

window对象

Global对象是作为 window对象的一部分加以实现的，可以在全局执行环境下获取 this 的值从而获取 Global 对象

Math

对象属性

  • Math.E 自然对数的底数，常量e的值
  • Math.LN10 10的自然对数
  • Math.LN2 2的自然对数
  • Math.LOG2E 以2为底e的对数
  • Math.LOG10E 以10为底e的对数
  • Math.PI π
  • Math.SQRT1_2 1/2平方根
  • Math.SQRT2 2平方根

min() 和 max()

一般用于求数组中的最大最小数，推荐使用 apply() 的方式确定设置 this 的值，将数组作为第二个参数

舍入方法

  • Math.ceil() 向上舍入，往大的舍
  • Math.floor() 向下舍入，往小的舍
  • Math.round() 四舍五入

random()

  • Math.random()

此方法返回一个 0 ~ 1之间的随机数，在此基础上经过处理可以得到更大的随机数

值 = Math.floor(Math.random() * 可能值范围 + 第一个可能值)

其他方法

Math对象中海油许多其他数学计算方法，比如 Math.abs()，Math.pow()，Math.sin() 等等，需要的时候可以查表
