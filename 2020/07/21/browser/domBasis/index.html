节点层次

根据 w3c 的 dom 标准，所有html的内容都是节点

  • html 元素是元素节点
  • 文本是文本节点
  • 元素的属性是属性节点
  • 注释是注释节点

Node 类型

DOM中所有元素都是一个节点，整个文档也是一个节点，而每个节点都有一些相同的属性，以下的这些属性与方法只是一个语法，如果要变成实例还需要与documnet配合，并且这些方法几乎使用于所有的节点

节点属性

nodeName 和 nodeValue

对于元素来说，nodeName 往往是标签名，nodeValue 为 null

childNodes

每个节点都有这个属性，其中有一个 NodeList 对象保存着该节点下所有的子节点，虽然这个对象很像数组，但是它并不是 Array 的实例

parentNode

每个节点都有的属性，指向了它的父节点

innerHTML

节点的文本值

attributes

节点的属性节点

firstChild 和 lastChild

分别指向该节点的第一个子节点和最后一个子节点，也就是 childNodes 里面的第一个值和最后一个值

创建节点

通过下面的方式创建节点才能进行一系列的修改、插入操作

  • createAttribute() 创建属性节点
  • createElement() 创建元素节点
  • createTextNode() 创建文本节点

操作节点

appendChild()

用于向指定节点的子节点对象 childNodes 的末尾推入一个节点

  • 如果推入的节点已经存在，那么不管它在哪，都会变成最后一个子节点

insertBefore()

两个参数，参数1为插入节点，参数2为参照节点，它会插在参照节点之前

replaceChild()

两个参数，参数1为插入的节点，参数2为替换节点。它会删除原先的节点，在将新的节点插入

替换时，原节点的所有指针关系会复制到新节点

removeChild()

直接移出某个节点

cloneNode()

接受一个布尔参数；true 表示深复制，即该节点即它下面的整个子节点树； false 表示只复制它自己。

复制后得到的节点必须通过上述添加节点的方法推入，否则该节点就只是一个副本

document

获取文档请求信息

  • URL 获取地址
  • domain 获取域名

获取节点

  • getElementById() 获取 id 为某某的标签，如果有多个只会获取一个
  • getElementsByTagName() 获取标签名为某某的所有标签
  • getElementsByClassName() 获取 class 为某某的所有标签，该方法在 IE 5，6，7，8 无效

通过上述三种方式获取节点，才能进行上面讲过的属性获取和方法的操作

除了id那种方式返回的都是数组，即使只有一个节点也是数组，所以进行节点操作时要用索引指明哪个元素，比如只有一个元素时就 -> document.getElementsByTagName(“body”)[0]

修改属性

HTML DOM 对象style属性

最常见的获取节点，并改变节点的样式

css中大部分属性都可以通过此方式修改

关于这一部分，可以到 W3C 官网查看完整的手册，各种边框、背景色、宽高、滚动条等等属性调用的方法。

W3C dom对象 style属性:

事实上，dom对象包括的东西很多，除了这些样式，还有许多事件属性，都是可以通过JS来改变的。网页的动态效果就是DOM对象提供的各种API，能让JS动态修改并在网页中反映出来，通过这样的方式实现绚丽多彩的网页效果和交互
