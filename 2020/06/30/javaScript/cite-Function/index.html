深入理解函数 Function
2020-06-30|JavaScript

Function

进一步理解函数

函数其实也是对象，每个函数都是Function的一个实例，因此函数名其实也是一个指针，下面的引用方式都是对的

另外，第三种构造方式会解析两次代码，作为变量的形式解析一次，传入构造函数的字串又解析一次

没有重载

其实知道函数是对象以后也就不难理解了，同样的函数名存的指针值都指向了同一个函数实例，就算写了多个函数名相同的函数，也不过是对象多次赋值一样，改变的不过是函数体的内容而已。最后一次赋值(写在最后的同名函数)，也就作为了最终生效的函数

函数声明与函数表达式

函数声明提升

ES里有个叫做函数声明提升的过程，只对函数声明方式的函数有效，表达式的方式无效

具体过程就是解析器在执行代码以前，将声明的函数放在源代码树的顶部，个人认为这样做是为了在任何地方调用函数时，能在作用域链中找到，而函数表达式的方式，它是某个变量引用了函数，在执行到这个变量之前，是解析不到函数的，况且没解析这个变量之前使用这个变量，要么找不到，要么可能会在父级作用域里找到同名的但其值绝不会是该函数。

  • 函数声明
  • 函数表达式

!!!注意，变量声明同样也有提升，而且低于函数声明，下面的例子，不管表达式声明的 gre 在函数声明的前还是后，都会覆盖函数声明

作为值

本身函数名作为变量，自然就可以作为值，当参数传递，或者是函数返回函数都可以。这里也对应前面 Array 的 sort() 方法，传入一个自定义比较函数

函数内部属性

arguments 和它的指针 callee

arguments 是储存函数参数的，而它还拥有一个属性 callee，用来指向拥有这个 arguments 的函数，在函数内部其实就是指向该函数本身

这样的好处在于，使用递归一类的计算方法时不会依赖于函数名，即使函数体被赋予给了其他变量，照样能正常使用

属性和方法

length 和 prototype

  • length 是函数接受到的参数个数
  • prototype 是一个很重要的属性，大部分的实例方法都保存在它的名下，第六章在详细理解

apply() call()

这两个方法并不是继承而来的，用途就是在特定的作用域调用函数，就是指定一个执行环境来调用函数，实际上就是设置 this 的值

  • apply 接受两个参数，第一个为指定的作用域，第二个为 Array实例或者 arguments对象

指定 this 之后就会到相应的执行环境找变量，若是找不到就为 undefined

  • call 与 apply 差异只是在第二个参数传递的方式，call 传递参数需要指定每一个参数，而 apply不用

bind()

它可以创建一个函数的实例，并指定这个函数的 this 值，而上面的 apply 和 call 一般是用于调用函数时指定其 this 值

《IE 9+，Firefox 4+，Safari 5.1+，Opera 12+，Chrome 支持 bind()方法》
