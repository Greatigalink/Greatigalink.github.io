变量

基本类型与引用类型

基本类型值是指一些简单的数据段，有五种类型，这种变量存储的就是值

  • 基本类型值是储存在栈内存的

  • Uundefined, Null, Number, Boolean, String

引用类型值是有多个值构成的对象 Object

  • 基本类型值是储存在堆内存的
  • Js不允许直接操作对象的内存空间，所以操作的是对象的引用
  • 其实引用的底层实现是创建一个指针指向对象的内存空间，这个指针只能指向该对象不能改变
  • 所以实际操作的时候是靠引用来间接赋值和修改的

定义及复制

定义及赋值

  • 基本类型直接赋值就行
  • 对象可以通过 new Object() 创建，以 Object.key = 13 的形势来添加新属性并赋值
  • 不允许对基本类型值进行如 a.key 这样添加新熟悉

复制变量

基本类型值其实就是创建了一个新变量，并把拷贝的值放进去，所以新旧变量完全独立，操作互不影响

引用类型复制是其实就是为同一个对象多添加了一个引用，所以新旧两个变量都是在操作一个对象，数据是双向同时变化的

参数传递

函数都是按值传递的

  • 基本类型的传递与其复制相同，外部传值与对应的局部变量为不同的值，互不影响

  • 对象传的也是值，只是这个值并不是真的值，而是指针值，看看下面的例子

看看下面的讲解

在讲对象是如何传值时，先谈谈什么是栈内存？堆内存？

栈内存往往是固定的大小，里面直接存放值，每个变量就像一块砖一样放在内存里，所以基本类型值存的就是值本身

而堆内存大小不固定，就像一堆砖一样，你可以只放一块，也可以放一百块，不像基本类型值大小固定

从这里就可以看出，对于基本类型值来说，存值本身最合适不过了，直接就能拿到数据。
但是引用类型值往往包含多个变量。难道我手里抱着一堆砖让你找？好像有点奇怪，还不如我告诉你那堆砖在哪，你自己去翻就好。
所以引用类型值存的是指针，指向那一组数据集合，你想要就去那堆砖找就好了，数据都在那堆砖里呢

所以不难看出，同样都是传值，基本值传的是值本身，引用传的是指针值；只不过引用传的指针值是指向一堆砖在哪，对于函数外定义的对象与函数传参对象，虽然变量名不一样，但指针值一样所以指向了同一堆砖，obj 修改，great 也变，但赋新对象给 obj 自然就指向了别的地方
